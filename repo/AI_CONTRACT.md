# AI Contract

このファイルは、このリポジトリでAIアシスタントと作業するためのルールを定義する。

## 基本方針

- 疑義があれば、実装や操作の前に質問する
- ユーザーの許可なく、ファイルの作成・更新・削除を行わない
- ユーザーの許可なく、依存追加・テスト実行・commit・pushを行わない
- プロジェクト管理下のファイルは、調査や説明のために読み取ってよい
- 要求がこのファイルの内容と矛盾する場合は、作業前に矛盾を指摘する

## 実装ルール

- 一度に大規模変更しない
- 変更は小さな単位に分ける
- 実装前に短い作業計画を提示する
- テスト無しで実装しない
- 既存の設計・命名・構成を優先する
- architecture変更時は事前に確認を取る

## コマンド実行ルール

- 読み取り専用の確認コマンドは実行してよい
  - 例: `git status`, `git diff`, `ls`, `find`, `rg`, `sed`, `cat`
- 状態を変更するコマンドは事前に許可を取る
  - 例: `git add`, `git commit`, `git push`, `pip install`, `pytest`, `rm`, ファイル生成・編集

## 進め方

- 追加要望を受けたら、まず意図・影響範囲・設計案を整理する
- 必要に応じて確認質問をする
- ユーザーが明示的に許可してから実装・テスト・commit・pushを行う
- 作業後は、変更内容・確認結果・未対応事項を簡潔に報告する
